# OverrideWorkFolder Parameter - Quick Reference

## What It Does

The `-OverrideWorkFolder` parameter allows you to specify a custom location where **all** output files from database creation will be stored, making it easier to:
- Collect logs for troubleshooting
- Archive complete operation records
- Organize outputs by project or date
- Avoid mixing outputs with default locations

## Usage

### Basic Usage
```powershell
# Create database with all outputs in custom folder
.\Db2-CreateInitialDatabases.ps1 `
    -OverrideWorkFolder "C:\MyOutputs\DB_Creation_$(Get-Date -Format 'yyyyMMdd_HHmmss')" `
    -DatabaseType PrimaryDb `
    -PrimaryInstanceName DB2
```

### Use Cases

#### 1. Project-Based Organization
```powershell
$projectFolder = "C:\Projects\DatabaseMigration\Outputs"
.\Db2-CreateInitialDatabases.ps1 -OverrideWorkFolder $projectFolder -DatabaseType BothDatabases
```

#### 2. Dated Archives
```powershell
$archiveFolder = "D:\Archives\$(Get-Date -Format 'yyyy-MM')\DB_Creation"
.\Db2-CreateInitialDatabases.ps1 -OverrideWorkFolder $archiveFolder -DatabaseType FederatedDb
```

#### 3. Troubleshooting
```powershell
$debugFolder = "C:\Troubleshooting\Issue_12345"
.\Db2-CreateInitialDatabases.ps1 -OverrideWorkFolder $debugFolder -DatabaseType PrimaryDb
```

## What Gets Stored

When you specify an override folder, **all** these files will be created there:

| File Type | Pattern | Description |
|-----------|---------|-------------|
| SQL Scripts | `*.sql` | Database creation and configuration SQL |
| BAT Scripts | `*.bat` | Batch files that execute SQL |
| Output Logs | `*.out` | Standard output from DB2 commands |
| Error Logs | `*.err` | Error output from DB2 commands |
| HTML Reports | `*.html` | Formatted execution reports |
| JSON Exports | `*.json` | Structured data exports |

### Typical Output Structure

```
C:\MyOutputs\DB_Creation_20251216_143022\
├── FKMTST\                                    # Database-specific subfolder
│   ├── 20251216-143025\                       # Timestamp subfolder
│   │   ├── Add-SpecificGrants_*.bat
│   │   ├── Add-SpecificGrants_*.sql
│   │   ├── Add-SpecificGrants_*.out
│   │   ├── Add-SpecificGrants_*.err
│   │   ├── Restart-InstanceAndActivateDatabase_*.bat
│   │   ├── Set-DatabasePermissions_*.bat
│   │   ├── New-Database_*.bat
│   │   ├── Db2-CreateInitialDatabase_FKMTST.html
│   │   └── Db2-AutoGeneratedCatalogsScripts_*.html
```

## Default Behavior (Without Override)

If you **don't** specify `-OverrideWorkFolder`, files go to:
```
C:\opt\data\Db2-CreateInitialDatabases\<DatabaseName>\<Timestamp>\
```

## Common Patterns

### Network Share Storage
```powershell
$networkShare = "\\fileserver\DatabaseLogs\$(Get-Date -Format 'yyyy-MM-dd')"
if (Test-Path $networkShare) {
    .\Db2-CreateInitialDatabases.ps1 -OverrideWorkFolder $networkShare -DatabaseType BothDatabases
}
```

### USB Drive Export
```powershell
$usbDrive = "E:\DB_Exports\Creation_$(Get-Date -Format 'yyyyMMdd_HHmm')"
New-Item -ItemType Directory -Path $usbDrive -Force
.\Db2-CreateInitialDatabases.ps1 -OverrideWorkFolder $usbDrive -DatabaseType PrimaryDb
```

### Temporary Testing
```powershell
$tempTest = Join-Path $env:TEMP "DBTest_$(Get-Random)"
.\Db2-CreateInitialDatabases.ps1 -OverrideWorkFolder $tempTest -DatabaseType FederatedDb
# Review outputs, then clean up
Remove-Item $tempTest -Recurse -Force
```

## Tips

1. **Always use full paths** - Avoid relative paths to prevent confusion
2. **Include timestamps** - Makes it easy to identify when operations ran
3. **Check disk space** - Database operations can generate large log files
4. **Use descriptive names** - Include database name or purpose in folder name
5. **Document your structure** - Keep a README in your output location

## Verification

After running, verify all files are in the expected location:

```powershell
$outputFolder = "C:\YourOutputFolder"

# List all files by type
Get-ChildItem -Path $outputFolder -Recurse -File | 
    Group-Object Extension | 
    Select-Object Name, Count | 
    Sort-Object Name | 
    Format-Table

# Check total size
$size = (Get-ChildItem -Path $outputFolder -Recurse -File | Measure-Object -Property Length -Sum).Sum
Write-Host "Total size: $([math]::Round($size / 1MB, 2)) MB"

# Find most recent files
Get-ChildItem -Path $outputFolder -Recurse -File | 
    Sort-Object LastWriteTime -Descending | 
    Select-Object -First 10 | 
    Format-Table Name, LastWriteTime, @{N='Size (KB)';E={[math]::Round($_.Length / 1KB, 2)}}
```

## Troubleshooting

### Problem: Files still appearing in C:\opt\data

**Solution**: Make sure you're using the parameter correctly:
```powershell
# ✅ Correct
.\Db2-CreateInitialDatabases.ps1 -OverrideWorkFolder "C:\MyOutput"

# ❌ Wrong - parameter name typo
.\Db2-CreateInitialDatabases.ps1 -OverideWorkFolder "C:\MyOutput"

# ❌ Wrong - no parameter
.\Db2-CreateInitialDatabases.ps1 "C:\MyOutput"
```

### Problem: Access denied creating folder

**Solution**: Ensure:
1. You have write permissions to the target location
2. The parent folder exists
3. You're running with appropriate privileges

```powershell
# Test write access first
$testFolder = "C:\MyOutput"
try {
    New-Item -ItemType Directory -Path $testFolder -Force -ErrorAction Stop
    Write-Host "✅ Folder accessible" -ForegroundColor Green
} catch {
    Write-Host "❌ Cannot write to folder: $_" -ForegroundColor Red
}
```

### Problem: Some files in override, some in default location

**Status**: This was a **bug** that has been **fixed**. If you still see this after the fix:
1. Verify you have the latest version of the scripts
2. Check that no other scripts are setting `Set-OverrideAppDataFolder` 
3. Review the detailed analysis in `OverrideWorkFolder-Analysis.md`
