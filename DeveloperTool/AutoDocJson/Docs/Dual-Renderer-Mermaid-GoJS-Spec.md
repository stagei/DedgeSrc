# Dual Renderer Specification: Mermaid / GoJS Switchable Diagram Rendering

## Overview

Add a renderer dropdown to all diagram views in AutoDocJson, allowing the user to switch between **Mermaid** (current SVG-based rendering) and **GoJS** (interactive, zoomable, clickable nodes). The Mermaid text already stored in JSON (`FlowMmd`, `SequenceMmd`, `ProcessMmd`, etc.) is parsed client-side into a Universal Graph Model (nodes + edges), then dispatched to either renderer.

## Motivation

| Concern | Mermaid (current) | GoJS (proposed alternative) |
|---|---|---|
| Rendering | Black-box: text → SVG, no control over layout | Full programmatic control over layout and interaction |
| Interactivity | Post-render hacks (MutationObserver, foreignObject fixes, manual click handlers on SVG DOM) | Native click, hover, drag, tooltip, selection built-in |
| Performance | Struggles above ~300 nodes; hangs browser on large COBOL programs | Handles 1000+ nodes with virtualized rendering |
| Pan/zoom | Requires svg-pan-zoom library + post-render init | Built-in pan, zoom, scroll-to-zoom |
| Dark mode | Requires `applyDarkModeLineStyles()` post-fix (edge colors, node fills, text) | Theme applied via node/link templates directly |
| Tab re-render | Must re-invoke `mermaid.run()` when Bootstrap tab becomes visible | Diagram updates on `requestUpdate()` when container resizes |

GoJS does not replace Mermaid -- both renderers remain available. The user picks via a dropdown per diagram. Preference is persisted in `localStorage`.

## Current Architecture

```
C# Parser → JSON (FlowMmd = raw Mermaid text string)
     ↓
Razor View → <pre class="mermaid">@Html.Raw(FlowMmd)</pre>
     ↓
Browser: mermaid.initialize({ startOnLoad: true }) → auto-renders <svg>
     ↓
autodoc-diagram-controls.js → post-fix (foreignObject, transforms, dark mode lines, pan-zoom)
```

## Proposed Architecture

```
C# Parser → JSON (FlowMmd = raw Mermaid text string)
     ↓
Razor View → <pre class="mermaid-source" style="display:none">@Html.Raw(FlowMmd)</pre>
           + <div class="diagram-render-target"></div>
           + Renderer dropdown [Mermaid ▾ | GoJS]
     ↓
Browser: autodoc-renderer.js
     ├─ parseMermaidToGraph(text)  → { nodes[], edges[] }  (Universal Graph Model)
     ├─ renderMermaid(ugm, target) → mermaid.render() as today
     └─ renderGoJS(ugm, target)   → go.Diagram with node/link templates
```

## Mermaid Text Parser Specification

### Why Parsing Is Feasible

All Mermaid text in AutoDocJson is **machine-generated** by `MermaidWriter.cs` and the individual parsers. It follows strict, predictable patterns with no hand-written edge cases.

### Node Shape Patterns

The parsers generate exactly these node shapes:

| Mermaid Syntax | Meaning | Regex | GoJS Figure | Color |
|---|---|---|---|---|
| `id[[label]]` | Program (stadium) | `(\w+)\[\[([^\]]+)\]\]` | `RoundedRectangle` | `#4f8cff` |
| `id((label))` | Section/paragraph (circle) | `(\w+)\(\(([^\)]+)\)\)` | `Circle` | `#34d399` |
| `id(label)` | Process (rounded rect) | `(\w+)\(([^\)]+)\)` | `RoundedRectangle` | `#64748b` |
| `id[(label)]` | Table (cylinder) | `(\w+)\[\(([^\)]+)\)\]` | `Database` | `#a78bfa` |
| `id[/label/]` | File (parallelogram) | `(\w+)\[\/([^\/]+)\/\]` | `Document` | `#fb923c` |
| `id[label]` | Generic (rectangle) | `(\w+)\[([^\]]+)\]` | `Rectangle` | `#94a3b8` |

### Edge Patterns

| Mermaid Syntax | Regex | Notes |
|---|---|---|
| `A --"label"-->B` | `(\S+)\s*--"([^"]*)"-->(.+)` | Quoted label |
| `A --label-->B` | `(\S+)\s*--(\w[\w\s]*)-->(.+)` | Unquoted label |
| `A -->B` | `(\S+)\s*-->(.+)` | No label |
| `A ---->B` | `(\S+)\s*-{3,}->(.+)` | Long arrow (same as regular edge) |
| `A ===>B` | `(\S+)\s*==+>(.+)` | Thick arrow |
| `A --->|label|B` | `(\S+)\s*-+>\|([^\|]*)\|(.+)` | Pipe-delimited label |

### Sequence Number Injection

`MermaidWriter.WriteLine()` injects `(#N)` before `-->` arrows. The parser must strip these:

```
Original:  init_section(#1)--"perform "-->read_next
Cleaned:   init_section --"perform "-->read_next
Regex:     \(#\d+\)
```

### Lines to Skip

| Pattern | Reason |
|---|---|
| `%%{ init: ... }%%` | Mermaid config directive |
| `flowchart TD` / `flowchart LR` | Direction declaration (captured as metadata) |
| `classDef ...` | Style definition (map to GoJS colors instead) |
| `class id1,id2 className` | Class assignment (use for color mapping) |
| `participant ...` | Sequence diagram syntax |
| Empty / whitespace-only lines | Padding |

### Parser Pseudocode

```javascript
function parseMermaidToGraph(mermaidText) {
  const nodes = new Map();
  const edges = [];
  let direction = 'TD';

  for (const line of mermaidText.split('\n')) {
    const trimmed = line.trim();

    // Skip non-graph lines
    if (!trimmed || trimmed.startsWith('%%') || trimmed.startsWith('classDef')
        || trimmed.startsWith('class ') || trimmed.startsWith('participant'))
      continue;

    // Capture direction
    const dirMatch = trimmed.match(/^flowchart\s+(LR|TD|RL|BT|TB)/);
    if (dirMatch) { direction = dirMatch[1]; continue; }

    // Strip sequence numbers: (#N)
    const cleaned = trimmed.replace(/\(#\d+\)/g, '');

    // Try to match edge pattern: A --"label"--> B[shape]
    const edgeParts = splitEdge(cleaned);
    if (edgeParts) {
      const fromNode = parseNodeShape(edgeParts.from);
      const toNode = parseNodeShape(edgeParts.to);
      if (fromNode) nodes.set(fromNode.id, fromNode);
      if (toNode) nodes.set(toNode.id, toNode);
      edges.push({
        from: fromNode?.id || edgeParts.fromId,
        to: toNode?.id || edgeParts.toId,
        label: edgeParts.label
      });
      continue;
    }

    // Standalone node definition
    const node = parseNodeShape(cleaned);
    if (node) nodes.set(node.id, node);
  }

  return { nodes: [...nodes.values()], edges, direction };
}
```

## Universal Graph Model (UGM)

The intermediate representation between parsing and rendering:

```javascript
{
  direction: "TD",                    // from "flowchart TD"
  nodes: [
    {
      id: "bsautos",                  // Mermaid node ID
      label: "BSAUTOS",              // Display text
      shape: "stadium",              // Semantic shape name
      goFigure: "RoundedRectangle",  // GoJS figure mapping
      color: "#4f8cff",              // Presentation color
      mermaidClass: "prog"           // Original classDef (if any)
    }
  ],
  edges: [
    {
      from: "bsautos",               // Source node ID
      to: "main_logic",              // Target node ID
      label: "initiated",            // Edge label text
      seq: 1                         // Original sequence number (stripped from arrow)
    }
  ]
}
```

### Shape-to-Renderer Mapping

| UGM Shape | GoJS Figure | GoJS Defaults | Mermaid (roundtrip) |
|---|---|---|---|
| `stadium` | `RoundedRectangle` | strokeWidth: 2, stroke: blue | `[[label]]` |
| `circle` | `Circle` | strokeWidth: 2, stroke: green | `((label))` |
| `rounded` | `RoundedRectangle` | strokeWidth: 1, stroke: slate | `(label)` |
| `cylinder` | `Database` | strokeWidth: 2, stroke: purple | `[(label)]` |
| `parallelogram` | `Document` | strokeWidth: 2, stroke: orange | `[/label/]` |
| `rectangle` | `Rectangle` | strokeWidth: 1, stroke: gray | `[label]` |

### localStorage Persistence

```javascript
const RENDERER_KEY = 'autodoc-diagram-renderer';

function getPreferredRenderer() {
  return localStorage.getItem(RENDERER_KEY) || 'mermaid';
}

function setPreferredRenderer(value) {
  localStorage.setItem(RENDERER_KEY, value);
}
```

When a dropdown changes, all diagrams on the page switch to the selected renderer and the preference is saved.

## New JavaScript Module: `autodoc-renderer.js`

### Responsibilities

| Function | Purpose |
|---|---|
| `parseMermaidToGraph(text)` | Parse Mermaid text → UGM |
| `renderWithMermaid(ugm, targetEl)` | Serialize UGM back to Mermaid text, call `mermaid.render()` |
| `renderWithGoJS(ugm, targetEl)` | Create `go.Diagram` with node/link templates from UGM |
| `switchRenderer(diagramType, renderer)` | Switch a diagram between renderers |
| `initDiagrams()` | On page load, find all `.diagram-wrapper`, parse sources, render with preferred renderer |

### GoJS Node Template

```javascript
function createGoJSDiagram(targetEl, ugm) {
  const colorMap = {
    stadium: '#4f8cff', circle: '#34d399', rounded: '#64748b',
    cylinder: '#a78bfa', parallelogram: '#fb923c', rectangle: '#94a3b8'
  };

  const layoutMap = {
    TD: { class: go.LayeredDigraphLayout, opts: { direction: 90, layerSpacing: 50 } },
    LR: { class: go.LayeredDigraphLayout, opts: { direction: 0, layerSpacing: 60 } },
    RL: { class: go.LayeredDigraphLayout, opts: { direction: 180, layerSpacing: 60 } },
    BT: { class: go.LayeredDigraphLayout, opts: { direction: 270, layerSpacing: 50 } }
  };

  const lc = layoutMap[ugm.direction] || layoutMap.TD;

  const diagram = new go.Diagram(targetEl, {
    layout: new lc.class(lc.opts),
    initialAutoScale: go.AutoScale.Uniform,
    padding: 20
  });

  diagram.nodeTemplate = new go.Node('Auto', {
    toolTip: go.GraphObject.build('ToolTip').add(
      new go.TextBlock({ margin: 4 }).bind('text', 'label')
    )
  }).add(
    new go.Shape({ strokeWidth: 2 })
      .bind('figure', 'goFigure')
      .bind('fill', 'color')
      .bind('stroke', 'color', c => shadeColor(c, -30)),
    new go.TextBlock({
      margin: 8, font: '12px Segoe UI', stroke: '#e2e4ea',
      maxSize: new go.Size(180, NaN), wrap: go.Wrap.Fit
    }).bind('text', 'label')
  );

  diagram.linkTemplate = new go.Link({
    routing: go.Routing.AvoidsNodes, corner: 8
  }).add(
    new go.Shape({ strokeWidth: 1.2 }).bind('stroke', 'color'),
    new go.Shape({ toArrow: 'Standard', scale: 1 })
      .bind('fill', 'color').bind('stroke', 'color'),
    new go.Panel('Auto').add(
      new go.Shape({ fill: '#242838', stroke: null }),
      new go.TextBlock({
        font: '10px Segoe UI', stroke: '#8b90a5', margin: 2
      }).bind('text', 'label')
    )
  );

  diagram.model = new go.GraphLinksModel({
    nodeKeyProperty: 'id',
    linkFromKeyProperty: 'from',
    linkToKeyProperty: 'to',
    nodeDataArray: ugm.nodes,
    linkDataArray: ugm.edges.map(e => ({
      ...e, color: colorMap[ugm.nodes.find(n => n.id === e.from)?.shape] || '#8b90a5'
    }))
  });

  return diagram;
}
```
