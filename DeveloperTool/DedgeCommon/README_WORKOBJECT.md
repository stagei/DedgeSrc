**Version:** DedgeCommon v1.5.20  
**Status:** Production Ready

---

## 🎯 What is WorkObject?

WorkObject is a dynamic data container that lets you:
- ✅ Accumulate data during program execution
- ✅ Track script/query executions with timestamps
- ✅ Export to JSON for programmatic access
- ✅ Export to beautiful HTML reports with tabs and Monaco editor
- ✅ Publish reports to web for team sharing

**Works in both C# and PowerShell with identical results!**

---

## 🚀 Quick Start - C#

```csharp
using DedgeCommon;

// 1. Create WorkObject
var workObject = new WorkObject();

// 2. Add properties
workObject.SetProperty("DatabaseName", "BASISPRO");
workObject.SetProperty("Server", "p-no1fkmprd-db");
workObject.SetProperty("Success", true);
workObject.SetProperty("RecordCount", 547);

// 3. Track script executions
workObject.AddScriptExecution(
    "Backup Database",
    "BACKUP DATABASE BASISPRO TO E:\\backups",
    "Completed in 145 seconds");

workObject.AddScriptExecution(
    "Verify Backup",
    "RESTORE DATABASE VERIFY ONLY",
    "Verification successful");

// 4. Export to JSON
var exporter = new WorkObjectExporter();
exporter.ExportToJson(workObject, @"C:\reports\backup.json");

// 5. Export to HTML with tabs and Monaco editor
exporter.ExportToHtml(
    workObject,
    @"C:\reports\backup.html",
    title: "Database Backup Report",
    autoOpen: true);  // Opens in browser automatically
```

## 🚀 Quick Start - PowerShell

```powershell
Import-Module GlobalFunctions

# 1. Create WorkObject
$workObject = [PSCustomObject]@{
    DatabaseName = "BASISPRO"
    Server = "p-no1fkmprd-db"
    Success = $true
    RecordCount = 547
    ScriptArray = @()
}

# 2. Track script executions
$workObject = Add-ScriptAndOutputToWorkObject -WorkObject $workObject `
    -Name "Backup Database" `
    -Script "BACKUP DATABASE BASISPRO TO E:\backups" `
    -Output "Completed in 145 seconds"

$workObject = Add-ScriptAndOutputToWorkObject -WorkObject $workObject `
    -Name "Verify Backup" `
    -Script "RESTORE DATABASE VERIFY ONLY" `
    -Output "Verification successful"

# 3. Export to JSON
Export-WorkObjectToJsonFile -WorkObject $workObject `
    -FileName "C:\reports\backup.json"

# 4. Export to HTML with tabs and Monaco editor
Export-WorkObjectToHtmlFile -WorkObject $workObject `
    -FileName "C:\reports\backup.html" `
    -Title "Database Backup Report" `
    -AutoOpen $true
```

## 📄 Template System

### Shared Template File:
```
C:\opt\src\DedgeSrc\DedgeSystemTools\Folders\DedgeCommon\Configfiles\Resources\HtmlTemplate.html
```

### Placeholders:
- `{{TITLE}}` - Page title
- `{{CONTENT}}` - Main content (generated by C# or PowerShell)
- `{{ADDITIONAL_STYLE}}` - Optional custom CSS

### How It Works:

**C# (HtmlTemplateService):**
1. Tries to load from Resources folder
2. Falls back to built-in template if file not found
3. Replaces placeholders
4. Returns complete HTML

**PowerShell (Get-HtmlTemplate):**
1. Tries to load from Resources folder
2. Falls back to built-in template if file not found
3. Replaces placeholders
4. Returns complete HTML

**Result:** Both use the SAME template file = consistent output!

## 🔍 Advanced Features

### Web Publishing:
```csharp
// C#: Publish to DevTools web path
exporter.ExportToHtml(
    workObject,
    @"C:\reports\backup.html",
    title: "Backup Report",
    addToDevToolsWebPath: true,           // Publish to web
    devToolsWebDirectory: "BackupReports", // Subdirectory
    autoOpen: true);

// Result: http://server/DevTools/BackupReports/backup.html
```

```powershell
# PowerShell: Publish to DevTools web path
Export-WorkObjectToHtmlFile -WorkObject $workObject `
    -FileName "C:\reports\backup.html" `
    -Title "Backup Report" `
    -AddToDevToolsWebPath $true `
    -DevToolsWebDirectory "BackupReports" `
    -AutoOpen $true

# Result: http://server/DevTools/BackupReports/backup.html
```

### Appending to Existing Scripts:
```csharp
// First execution
workObject.AddScriptExecution("DB Check", "SELECT COUNT(*)", "547");

// Later execution (appends to same script)
workObject.AddScriptExecution("DB Check", "SELECT MAX(ID)", "12345");

// Result: One tab "DB Check" with both executions and timestamps
```
