## 📝 Documentation Maintenance

### When to Update Docs:
- **README.md** - When adding/changing public API
- **RELEASE_NOTES_vX.Y.Z.md** - Create ONE new file per version
- **PROJECT_STATUS.md** - When project status changes
- **Architecture docs** - Only for significant design changes

### When NOT to Create Docs:
- ❌ After completing each task/fix
- ❌ Session summaries
- ❌ Implementation logs
- ❌ Multiple release notes per version
- ❌ Duplicate information

**See `.cursorrules` for detailed documentation standards.**
