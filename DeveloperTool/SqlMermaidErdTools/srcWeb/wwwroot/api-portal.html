🚀 Quick Start

1. Get Your API Key

After purchasing a Pro or Enterprise license, generate your API key below.

2. Make Your First Request

cURL 
curl -X POST "http://localhost:5001/api/v1/conversion/sql-to-mermaid" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY_HERE" \
  -d '{
    "sql": "CREATE TABLE users (id INT PRIMARY KEY, name VARCHAR(100));",
    "includeAst": false
  }'
JavaScript 
const response = await fetch('http://localhost:5001/api/v1/conversion/sql-to-mermaid', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'YOUR_API_KEY_HERE'
  },
  body: JSON.stringify({
    sql: 'CREATE TABLE users (id INT PRIMARY KEY, name VARCHAR(100));',
    includeAst: false
  })
});

const result = await response.json();
console.log(result.result); // Mermaid diagram
Python 
import requests

response = requests.post(
    'http://localhost:5001/api/v1/conversion/sql-to-mermaid',
    headers={'X-API-Key': 'YOUR_API_KEY_HERE'},
    json={
        'sql': 'CREATE TABLE users (id INT PRIMARY KEY, name VARCHAR(100));',
        'includeAst': False
    }
)

result = response.json()
print(result['result'])  # Mermaid diagram

📡 Available Endpoints

POST

/api/v1/conversion/sql-to-mermaid

Convert SQL DDL to Mermaid ERD diagram

Body: { sql: string, includeAst?: boolean }
POST

/api/v1/conversion/mermaid-to-sql

Convert Mermaid ERD to SQL DDL

Body: { mermaid: string, dialect?: string, includeAst?: boolean }
Dialects: AnsiSql, SqlServer, PostgreSql, MySql
POST

/api/v1/conversion/generate-migration

Generate SQL migration from Mermaid diagram differences

Body: { beforeMermaid: string, afterMermaid: string, dialect?: string }
GET

/api/v1/auth/key-info

Get information about your API key (usage, limits, tier)

Headers: X-API-Key: YOUR_KEY
