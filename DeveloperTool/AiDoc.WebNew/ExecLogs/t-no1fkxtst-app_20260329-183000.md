## Code Changes (v1.0.13)

### `python/server_http.py`

1. **`_check_dependencies()`** — new startup function that eagerly imports `chromadb` and `onnxruntime` before the HTTP server starts listening. If either is missing, logs a clear error and returns exit code 1 so NSSM sees the crash immediately (instead of serving a misleading `/health OK` while queries fail).

2. **`/ready` endpoint** — new deep readiness check in `RAGHandler.do_GET`. Calls `_get_collection()` to verify the ChromaDB collection is actually openable. Returns `{"ready": true, "rag": "..."}` on success or `{"ready": false, "error": "..."}` with HTTP 503 on failure. Complements `/health` (liveness-only) for real readiness probing.

### `scripts/Fix-VenvAndRestart.ps1`

Updated post-fix health check from `GET /health` to `GET /ready`. The script now verifies the venv is genuinely functional (can open ChromaDB), not just that the process is alive.

### `AiDoc.WebNew.csproj`

Removed the `PostBuild` and `PostPublishSign` MSBuild targets that were signing the DLL/EXE inline during `dotnet publish`. These used a stale FQDN (`dedge-server.DEDGE.fk.no`) that caused `dotnet publish` to always fail in Release mode. Signing is already handled in `Build-And-Publish.ps1` after publish.

### `Build-And-Publish.ps1`

Fixed DedgeSign path from FQDN `dedge-server.DEDGE.fk.no` to short name `dedge-server`.
